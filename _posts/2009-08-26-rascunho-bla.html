Hoje a tarde conversando com uma pessoa ouvi a seguinte pergunta: Que vantagem que você teve em programar JavaScript de forma Orientada a Objetos se ela não tem essa característica? JavaScript é uma linguagem funcional, portanto você não deveria tentar usá-la como OO.

Essa pergunta foi no mínimo esquisita, já que os paradigmas Funcional e Orientado a Objetos não são necessáriamente excludentes. Elas podem perfeitamente ser usadas em conjunto. Na verdade, ele estava se referindo as High-Order Functions presentes em JavaScript. Porém, isso é apenas uma parte do paradigma funcional. Existem outras características como imutabilidade, ausência de operações destrutivas, ausência de estado, dentre outras, que caracterizam a programação funcional (resolvendo um bocado de problemas de concorrência, diga-se de passagem). Isso é descrito bem na wikipedia e no  paper de John Hughes.

A conversa proseguiu e surgiu outra frase dizendo que Ruby e Python eram linguagem funcionais. Mais uma vez fiquei bem surpreso. Primeiro porque ruby é uma das linguagens que mais bem implementa OO de forma exemplarmente bonita. Python da mesma forma é uma linguagem orientada a objetos e basta pesquisar um pouco para saber disso. Mais uma vez o colega estava se referindo ao suporte de High-Order Functions apenas.

Papo vai, papo vem e surge um exemplo dado pelo colega desse colega de como a programação funcional resolveu um problema de IoC em Python (ahm??). Nessa hora meus cabelos quase cairam da minha cabeça.  Na verdade ele estava querendo dizer que como a linguagem é dinâmica, resolver o problema é mais simples que a maneira com que resolvemos em Java ou C#.

Passado um tempo eu questionei sobre a ausência de objetos e o tal uso das simples funções separadas apenas em arquivos(segundo a sugestão do colega): Mas como vou organizar essas funções? Não seria legal no mínimo colocá-las em namespaces, mesmo que seja apenas para evitar colisão de nomes? A resposta que obtive foi: Se houver colisão de nomes os testes pegarão. :(

Nesse momento já não valia mais a pena discutir. Não existia mais a possibilidade da conversa evoluir. Não estavamos falando o mesmo idioma. Não tinha como se comunicar.

Pra finalizar, acho que vale a pena ler o artigo do Phillip Calçado, o artigo do Carlos Villela e ver uma apresentação do Uncle Bob para tirar algumas dúvidas sobre Orientação a Objetos e JavaScript.
